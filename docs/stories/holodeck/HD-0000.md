# HD-0000 — BIC Data Models

**Phase:** 0 — Foundation
**Status:** Pre-Approved

---

## Goal

Create `apps/bic/` as a standalone uv workspace member containing all normalized Broker
Interface Contract (BIC) data models. No abstract class yet — pure data models first.
Both `holodeck` and future `K9` will import from this package.

---

## New Files

| File | Purpose |
|---|---|
| `apps/bic/pyproject.toml` | Workspace member; no runtime deps; hatchling build |
| `apps/bic/src/bic/__init__.py` | Re-exports all 9 model classes |
| `apps/bic/src/bic/models.py` | All BIC `@dataclass` models |

### 4. Write `src/bic/models.py`

Use `from __future__ import annotations` at the top. Import only `dataclasses`, `datetime`.
All fields use plain Python types — no Pydantic, no enums.

```python
from __future__ import annotations
from dataclasses import dataclass, field
from datetime import date, datetime


@dataclass
class AccountSnapshot:
    account_id: str
    net_liquidation: float
    available_funds: float
    buying_power: float


@dataclass
class Position:
    symbol: str
    quantity: int
    avg_price: float
    position_type: str  # e.g. "OPTION", "STOCK"


@dataclass
class Quote:
    symbol: str
    last: float
    bid: float
    ask: float


@dataclass
class OptionContract:
    strike: float
    option_type: str  # "CALL" or "PUT"
    bid: float
    ask: float
    delta: float


@dataclass
class OptionChain:
    symbol: str
    expiration: date
    options: list[OptionContract] = field(default_factory=list)


@dataclass
class OrderLeg:
    action: str       # "BUY" or "SELL"
    option_type: str  # "CALL" or "PUT"
    strike: float
    expiration: date


@dataclass
class OrderRequest:
    symbol: str
    strategy_type: str   # e.g. "IRON_CONDOR", "PUT_CREDIT_SPREAD"
    legs: list[OrderLeg] = field(default_factory=list)
    quantity: int = 1
    order_type: str = "LIMIT"
    limit_price: float = 0.0


@dataclass
class OrderResponse:
    order_id: str
    status: str   # "ACCEPTED" or "REJECTED"


@dataclass
class Order:
    order_id: str
    status: str              # "OPEN", "FILLED", "CANCELED"
    filled_price: float | None = None
    remaining_quantity: int = 0
```

### 6. Write `tests/test_models.py`

Nine tests, one per model class:

```python
from datetime import date, datetime
from bic.models import (
    AccountSnapshot, Order, OrderLeg, OrderRequest, OrderResponse,
    OptionChain, OptionContract, Position, Quote,
)

def test_account_snapshot():
    s = AccountSnapshot("acct-1", 100_000.0, 80_000.0, 50_000.0)
    assert s.account_id == "acct-1"
    assert s.buying_power == 50_000.0

def test_position():
    p = Position("SPX", 1, 2.50, "OPTION")
    assert p.position_type == "OPTION"

def test_quote():
    q = Quote("SPX", 5800.0, 5799.95, 5800.05)
    assert q.bid < q.last < q.ask

def test_option_contract_put():
    c = OptionContract(5750.0, "PUT", 1.20, 1.30, -0.20)
    assert c.option_type == "PUT"
    assert c.delta == -0.20

def test_option_chain_defaults_empty():
    chain = OptionChain("SPX", date(2026, 1, 2))
    assert chain.options == []

def test_order_leg_sell():
    leg = OrderLeg("SELL", "PUT", 5750.0, date(2026, 1, 2))
    assert leg.action == "SELL"

def test_order_request_defaults():
    req = OrderRequest("SPX", "IRON_CONDOR")
    assert req.legs == []
    assert req.order_type == "LIMIT"
    assert req.quantity == 1

def test_order_response():
    r = OrderResponse("HD-000001", "ACCEPTED")
    assert r.status == "ACCEPTED"

def test_order_filled_price_default_none():
    o = Order("HD-000001", "OPEN")
    assert o.filled_price is None
    assert o.status == "OPEN"
```

## Notes

- `from __future__ import annotations` at top of `models.py` keeps forward references clean
- Do NOT use Pydantic — pure `@dataclass` per project convention
- `OptionChain.options` and `OrderRequest.legs` use `field(default_factory=list)` to avoid the mutable default trap
- String literals for status/action/option_type (no enums) keeps the interface minimal and serialization simple
