## Goal

Add historical bar data capability to the BIC interface so that visualization tools (and
future algorithmic strategies) can retrieve OHLCV candles from any broker implementation
in a uniform way.

Concretely:
- Add `OHLCVBar` dataclass to `bic.models`
- Add `get_ohlcv_bars(symbol, start, end, resolution)` abstract method to `bic.broker.Broker`
- Implement it in `HolodeckBroker` by aggregating the existing 1-minute synthetic data
- Expose a `MarketDataStore.get_bars_range()` helper for range queries

### 3. Add `get_bars_range()` to `MarketDataStore`

```python
def get_bars_range(
    self, start: datetime, end: datetime
) -> list[tuple[datetime, float]]:
    """Return (timestamp, last) pairs for all 1-minute bars in [start, end].

    start/end may be tz-aware; comparisons use naive local market time to match
    the CSV keys (which have no timezone suffix).
    Returns list sorted by timestamp ascending.
    """
    naive_start = start.replace(tzinfo=None) if start.tzinfo else start
    naive_end = end.replace(tzinfo=None) if end.tzinfo else end

    result: list[tuple[datetime, float]] = []
    for key, (last, _bid, _ask) in self._data.items():
        dt = datetime.fromisoformat(key)
        if naive_start <= dt <= naive_end:
            result.append((dt, last))
    result.sort(key=lambda x: x[0])
    return result
```

### 4. Implement `get_ohlcv_bars()` in `HolodeckBroker`

```python
def get_ohlcv_bars(
    self, symbol: str, start: datetime, end: datetime, resolution: str
) -> list[OHLCVBar]:
    if symbol != "SPX":
        raise ValueError(
            f"HolodeckBroker only supports SPX in MVP. Got: {symbol!r}"
        )
    valid = {"1m", "1h", "1d", "1w"}
    if resolution not in valid:
        raise ValueError(
            f"Unknown resolution {resolution!r}. Must be one of {sorted(valid)}."
        )

    raw = self._market_data.get_bars_range(start, end)
    if not raw:
        return []

    def _bar_key(dt: datetime) -> tuple:
        if resolution == "1m":
            return (dt.year, dt.month, dt.day, dt.hour, dt.minute)
        if resolution == "1h":
            return (dt.year, dt.month, dt.day, dt.hour)
        if resolution == "1d":
            return (dt.year, dt.month, dt.day)
        # "1w"
        iso = dt.isocalendar()
        return (iso[0], iso[1])  # (year, week)

    bars: list[OHLCVBar] = []
    for _key, group in itertools.groupby(raw, key=lambda x: _bar_key(x[0])):
        points = list(group)
        timestamps, prices = zip(*points)
        bars.append(
            OHLCVBar(
                symbol=symbol,
                timestamp=timestamps[0],
                open=prices[0],
                high=max(prices),
                low=min(prices),
                close=prices[-1],
            )
        )
    return bars
```

Add `import itertools` at the top of `broker.py`.

def test_ohlcv_1m_passthrough(full_broker):
    start = datetime(2026, 1, 2, 9, 30, tzinfo=ZoneInfo(TZ))
    end = datetime(2026, 1, 2, 10, 0, tzinfo=ZoneInfo(TZ))
    bars = full_broker.get_ohlcv_bars("SPX", start, end, "1m")
    assert len(bars) == 31   # 09:30–10:00 inclusive
    assert all(isinstance(b, OHLCVBar) for b in bars)

def test_ohlcv_1h_bar_count(full_broker):

    bars = full_broker.get_ohlcv_bars("SPX", start, end, "1h")

    end = datetime(2026, 1, 8, 15, 0, tzinfo=ZoneInfo(TZ))
    bars = full_broker.get_ohlcv_bars("SPX", start, end, "1d")
    assert len(bars) == 5   # Jan 2, 5, 6, 7, 8

## Notes

- `OHLCVBar.volume` is always 0 for Holodeck — the synthetic CSV has no volume column.
  A future Tradier implementation would populate this field.
- `open` of a bar = `last` price of the first minute bar in the period (not a true open tick,
  but accurate for our synthetic random-walk data where each bar's `last` is the price).
- `get_bars_range()` strips `tzinfo` before comparing against the CSV's naive timestamps.
  This keeps the comparison correct regardless of whether the caller passes tz-aware datetimes.
- Weekly bars use Python's `datetime.isocalendar()` ISO week number, so Jan 2 (a Friday in
  week 1) and Jan 5–9 (week 2) land in separate bars.
