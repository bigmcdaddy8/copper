## Tasks

### 1. Add time window check to `engine/runner.py`

Add this check immediately after step 5 (get current time), before the account check.
The `TradeSpec` does not currently have an `entry_window` field — add a simple
`earliest_entry_time` / `latest_entry_time` pair to the spec schema in K9-0010 if
desired, or use the run time alone.  For MVP, add an optional `allowed_entry_after`
and `allowed_entry_before` (HH:MM strings, CT) to `TradeSpec` and skip execution if the
current time is outside the window.

**Simplest MVP approach** — add optional fields to `TradeSpec` with sensible defaults:

In `config.py`, add to `TradeSpec`:

```python
    allowed_entry_after: str = "09:25"   # HH:MM CT — don't trade before this
    allowed_entry_before: str = "14:30"  # HH:MM CT — don't trade after this
```

In `runner.py`, add after getting `now`:

```python
        # Step 5b — execution time window check
        from zoneinfo import ZoneInfo
        ct = now.astimezone(ZoneInfo("America/Chicago"))
        after_h, after_m  = (int(x) for x in spec.allowed_entry_after.split(":"))
        before_h, before_m = (int(x) for x in spec.allowed_entry_before.split(":"))
        window_open  = ct.replace(hour=after_h,  minute=after_m,  second=0, microsecond=0)
        window_close = ct.replace(hour=before_h, minute=before_m, second=0, microsecond=0)
        if not (window_open <= ct <= window_close):
            result.outcome = "SKIPPED"
            result.reason = (
                f"Current time {ct.strftime('%H:%M')} CT is outside entry window "
                f"{spec.allowed_entry_after}–{spec.allowed_entry_before} CT."
            )
            return result
```

Also add `allowed_entry_after` and `allowed_entry_before` to `TradeSpec.from_json()`:

```python
            allowed_entry_after=data.get("allowed_entry_after", "09:25"),
            allowed_entry_before=data.get("allowed_entry_before", "14:30"),
```

### 2. Add daily entry guard to `engine/runner.py`

Add a `max_entries_per_day` guard using the run log directory.  After the position
check (step 7), count today's log files for this spec:

```python
        # Step 7b — max entries per day
        if spec.constraints.max_entries_per_day > 0:
            from K9.output.run_log import _LOG_DIR
            today_str = today.strftime("%Y%m%d")
            today_logs = list(_LOG_DIR.glob(f"{spec_name}_{today_str}_*.json"))
            if len(today_logs) >= spec.constraints.max_entries_per_day:
                result.outcome = "SKIPPED"
                result.reason = (
                    f"Already ran {len(today_logs)} time(s) today "
                    f"(max_entries_per_day={spec.constraints.max_entries_per_day})."
                )
                return result
```

### 3. Modify `src/K9/cli.py` — exit code reflects outcome

After writing the log, replace the existing exit logic with:

```python
    bad_outcomes = {"CANCELED", "REJECTED", "ERROR"}
    if result.outcome in bad_outcomes:
        raise typer.Exit(1)
    raise typer.Exit(0)
```

This makes K9 useful in shell scripts: `K9 enter --trade-spec ...` exits 0 on
FILLED or SKIPPED (normal business outcomes), exits 1 on CANCELED/REJECTED/ERROR
(something went wrong).

### 4. Write `tests/test_integration.py`

```python
"""End-to-end integration tests for K9 using HolodeckBroker (K9-0090)."""
from __future__ import annotations

import json
import subprocess
import sys
from pathlib import Path

import pytest


@pytest.fixture
def spec_file(tmp_path):
    """Write a minimal Holodeck trade spec to tmp_path."""
    data = {
        "enabled": True,
        "environment": "holodeck",
        "underlying": "SPX",
        "trade_type": "IRON_CONDOR",
        "wing_size": 5,
        "short_strike_selection": {"method": "DELTA", "value": 20},
        "position_size": {"mode": "fixed_contracts", "contracts": 1},
        "account_minimum": 100,
        "max_risk_per_trade": 1000,
        "minimum_net_credit": 0.01,
        "max_combo_bid_ask_width": 100.0,
        "entry": {
            "order_type": "LIMIT",
            "limit_price_strategy": "MID",
            "max_fill_time_seconds": 30,
        },
        "exit": {
            "take_profit_percent": 34,
            "expiration_day_exit_mode": "HOLD_TO_EXPIRATION",
        },
        "constraints": {"max_entries_per_day": 5, "one_position_per_underlying": False},
        "allowed_entry_after": "00:00",
        "allowed_entry_before": "23:59",
    }
    p = tmp_path / "test_ic.json"
    p.write_text(json.dumps(data))
    return p


def test_enter_exits_zero_or_one(spec_file, tmp_path):
    """K9 enter should exit 0 (FILLED/SKIPPED) or 1 (CANCELED/ERROR)."""
    result = subprocess.run(
        [
            sys.executable, "-m", "K9",
            "enter",
            "--trade-spec", "test_ic",
            "--specs-dir", str(spec_file.parent),
        ],
        capture_output=True,
        text=True,
        env={
            **__import__("os").environ,
            "K9_LOG_DIR": str(tmp_path / "logs"),
        },
    )
    assert result.returncode in (0, 1), (
        f"Unexpected exit code {result.returncode}\nstdout: {result.stdout}\nstderr: {result.stderr}"
    )


def test_enter_disabled_spec_exits_zero(spec_file, tmp_path):
    data = json.loads(spec_file.read_text())
    data["enabled"] = False
    spec_file.write_text(json.dumps(data))
    result = subprocess.run(
        [
            sys.executable, "-m", "K9",
            "enter",
            "--trade-spec", "test_ic",
            "--specs-dir", str(spec_file.parent),
        ],
        capture_output=True,
        text=True,
    )
    assert result.returncode == 0
    assert "disabled" in result.stdout.lower()


def test_enter_missing_spec_exits_one():
    result = subprocess.run(
        [sys.executable, "-m", "K9", "enter", "--trade-spec", "nonexistent_spec_xyz"],
        capture_output=True,
        text=True,
    )
    assert result.returncode == 1
```

## Notes

- The time window check uses `now.astimezone(ZoneInfo("America/Chicago"))` so it works
  correctly whether the host machine is in CT or any other timezone.
- The daily entry guard reads log files from disk.  This is simple and sufficient for MVP,
  since K9 is designed for single-machine cron execution.  A future story could use a
  database or lock file instead.
- `"allowed_entry_after": "00:00"` and `"allowed_entry_before": "23:59"` in the test spec
  ensure the time window never blocks test runs regardless of when they execute.
- `K9_LOG_DIR` env var override in the subprocess test keeps integration test logs in
  `tmp_path` — update `run_log.py` to check `os.environ.get("K9_LOG_DIR")` when
  constructing the default log directory.  Alternatively, pass `log_dir` through the
  runner.  Either approach is acceptable.
