### 4. Write `apps/K9/README.md`

```markdown
# K9 — Automated 0DTE Options Trade Entry

K9 is a command-line application that reads a JSON trade spec, validates market
conditions, and places a defined-risk 0DTE options strategy via the
**Broker Interface Contract (BIC)**.

## Environments

| `environment` | Broker | Requirements |
|---|---|---|
| `"holodeck"` | HolodeckBroker (local simulation) | None — uses synthetic Jan 2026 SPX data |
| `"sandbox"` | TradierBroker (paper trading) | `TRADIER_SANDBOX_API_KEY`, `TRADIER_ACCOUNT_ID` |
| `"production"` | TradierBroker (live) | `TRADIER_API_KEY`, `TRADIER_ACCOUNT_ID` |

## Quick Start

```bash
# Install dependencies
uv sync --all-groups --all-packages

# Run a Holodeck simulation (no API keys required)
uv run K9 enter --trade-spec spx_ic_20d_w5_tp34_0900

# Show help
uv run K9 enter --help
```

## Trade Specs

Trade specs live in `apps/K9/trade_specs/`.  Pass the file name without `.json`:

```bash
uv run K9 enter --trade-spec spx_pcs_20d_w5_tp50_0930
```

### Schema

| Field | Type | Description |
|---|---|---|
| `enabled` | bool | Set `false` to disable without deleting |
| `environment` | string | `"holodeck"` / `"sandbox"` / `"production"` |
| `underlying` | string | `"SPX"`, `"XSP"`, `"NDX"`, `"RUT"` |
| `trade_type` | string | `"IRON_CONDOR"`, `"PUT_CREDIT_SPREAD"`, `"CALL_CREDIT_SPREAD"` |
| `wing_size` | int | Points between short and long strike |
| `short_strike_selection.method` | string | `"DELTA"` only in MVP |
| `short_strike_selection.value` | float | Target delta (e.g. `20` for ~0.20Δ) |
| `account_minimum` | float | Required net liquidation ($) |
| `max_risk_per_trade` | float | Max loss allowed ($) |
| `minimum_net_credit` | float | Minimum acceptable net credit ($) |
| `max_combo_bid_ask_width` | float | Max sum of all leg spreads ($) |
| `entry.max_fill_time_seconds` | int | Seconds before order is canceled |
| `exit.take_profit_percent` | float | % of entry credit at which TP is placed |
| `allowed_entry_after` | string | Earliest entry time, CT (`"HH:MM"`) |
| `allowed_entry_before` | string | Latest entry time, CT (`"HH:MM"`) |

## Environment Variables (sandbox / production only)

```
TRADIER_API_KEY=<live token>
TRADIER_SANDBOX_API_KEY=<sandbox token>
TRADIER_ACCOUNT_ID=<account number>
```

Copy `.env.example` to `.env` and fill in your values.  Never commit `.env`.

## Exit Codes

| Code | Meaning |
|---|---|
| `0` | FILLED or SKIPPED (normal outcomes) |
| `1` | CANCELED, REJECTED, or ERROR |

## Run Logs

Every execution writes a JSON log to `logs/K9/<spec_name>_<YYYYMMDD_HHMMSS>.json`.

## Cron (example)

```cron
# Run SPX IC daily at 9:00 CT on weekdays
0 9 * * 1-5 cd /path/to/copper && uv run K9 enter --trade-spec spx_ic_20d_w5_tp34_0900
```

## Supported Trade Types

| Type | Legs |
|---|---|
| Iron Condor | Short put, long put, short call, long call |
| Put Credit Spread | Short put, long put |
| Call Credit Spread | Short call, long call |

## Notes

- The sandbox spec is disabled by default (`"enabled": false`) so a bare
  `K9 enter --trade-spec spx_ic_20d_w5_tp34_0900_sandbox` exits 0 with a "disabled"
  message rather than accidentally placing a paper trade.  Enable it manually when
  ready to test against the Tradier sandbox.
- `logs/` should be in `.gitignore` to prevent run log files from being committed.
- The README uses triple-backtick fences for code blocks — note that in the story the
  inner fences appear inside a larger fenced block, which is valid Markdown but may
  render oddly in some viewers.  The intent is clear from context.
