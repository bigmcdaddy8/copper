## Goal

the full position at the take-profit price.  This story adds `place_tp_order()` to
`engine/order.py` and integrates it into `engine/runner.py` as step 15 of the flow.

## Tasks

### 1. Add `build_tp_order()` to `engine/constructor.py`

```python
def build_tp_order(
    spec: TradeSpec,
    entry_order: OrderRequest,
    entry_fill_price: float,
) -> OrderRequest:
    """Build the GTC buy-to-close take-profit order.

    TP limit price = entry_fill_price × (1 - take_profit_percent / 100), rounded to 0.05.
    The TP order reverses each entry leg (SELL → BUY, BUY → SELL).

    Args:
        spec: TradeSpec containing exit configuration.
        entry_order: The original filled entry OrderRequest.
        entry_fill_price: The actual fill price (net credit received).

    Returns:
        A new OrderRequest representing the buy-to-close TP order.
    """
    tp_pct = spec.exit.take_profit_percent / 100.0
    raw_price = entry_fill_price * (1.0 - tp_pct)
    tp_price = round(round(raw_price / 0.05) * 0.05, 2)  # round to nearest $0.05

    reversed_legs = [
        OrderLeg(
            action="BUY" if leg.action == "SELL" else "SELL",
            option_type=leg.option_type,
            strike=leg.strike,
            expiration=leg.expiration,
        )
        for leg in entry_order.legs
    ]

    return OrderRequest(
        symbol=entry_order.symbol,
        strategy_type=entry_order.strategy_type,
        legs=reversed_legs,
        quantity=entry_order.quantity,
        order_type="LIMIT",
        limit_price=tp_price,
    )
```

Add `from bic.models import OrderLeg` to the imports at the top of `constructor.py`
(it was already imported via `OrderRequest` — confirm it's explicitly listed).

### 2. Add `place_tp_order()` to `engine/order.py`

```python
@dataclass
class TpOutcome:
    status: str     # "PLACED" | "FAILED"
    order_id: str = ""
    tp_price: float | None = None
    reason: str = ""


def place_tp_order(broker: Broker, tp_order: OrderRequest) -> TpOutcome:
    """Submit the take-profit GTC order.

    Returns a TpOutcome.  Does not poll — K9 MVP holds to expiration if TP
    is not hit; expiration is handled by the broker (Holodeck ExpirationEngine
    or Tradier automatic expiration).

    Args:
        broker: BIC Broker instance.
        tp_order: The buy-to-close OrderRequest (GTC limit).

    Returns:
        TpOutcome with status "PLACED" or "FAILED".
    """
    try:
        response = broker.place_order(tp_order)
        if response.status == "ACCEPTED":
            return TpOutcome(
                status="PLACED",
                order_id=response.order_id,
                tp_price=tp_order.limit_price,
            )
        return TpOutcome(
            status="FAILED",
            reason=f"Broker rejected TP order: {response.status}",
        )
    except Exception as exc:  # noqa: BLE001
        return TpOutcome(status="FAILED", reason=str(exc))
```

def test_build_tp_order_price(spec):
    from bic.models import OrderLeg, OrderRequest
    from datetime import date
    from K9.engine.constructor import build_tp_order

    entry_order = OrderRequest(
        "SPX", "IRON_CONDOR",
        legs=[
            OrderLeg("SELL", "PUT",  5800.0, date(2026, 1, 5)),
            OrderLeg("BUY",  "PUT",  5795.0, date(2026, 1, 5)),
            OrderLeg("SELL", "CALL", 5840.0, date(2026, 1, 5)),
            OrderLeg("BUY",  "CALL", 5845.0, date(2026, 1, 5)),
        ],
        limit_price=1.10,
    )
    spec.exit.take_profit_percent = 34
    tp_order = build_tp_order(spec, entry_order, filled_price=1.10)

    # TP price = 1.10 × (1 - 0.34) = 0.726 → rounded to nearest $0.05 = $0.70
    assert tp_order.limit_price == pytest.approx(0.70, abs=0.01)


def test_build_tp_order_reverses_legs(spec):
    from bic.models import OrderLeg, OrderRequest
    from datetime import date
    from K9.engine.constructor import build_tp_order

    entry_order = OrderRequest(
        "SPX", "IRON_CONDOR",

## Notes

- `"GTC"` (Good Till Cancel) duration is implied for the TP order: K9 MVP does not use
  intraday TP cancellation — if TP is not hit, the position expires.  `TradierBroker`
  must set `"duration": "gtc"` in the POST payload for TP orders.  This requires a
  small update to `TradierBroker.place_order()` to accept a `duration` parameter (or
  detect it from the strategy type).  The simplest approach: add an optional
  `duration: str = "day"` to `OrderRequest` in BIC, then K9-0030 uses it.
- TP price rounding uses `round(raw / 0.05) * 0.05` to snap to the nearest $0.05 tick.
  This is the minimum price increment for SPX options.
- `place_tp_order()` does **not** poll — K9 MVP is fire-and-forget for the TP order.
  The position will either hit TP during the session or expire at end of day.
- If TP placement fails, `run_entry()` records the error in `result.errors` but still
  returns `outcome="FILLED"` — the entry was successful even if TP placement failed.
  The run log (K9-0080) will capture the failure for review.
