# K9-0130 — Preflight Command + Dry-Run Mode

**Phase:** 7 — Reliability & YAML Cutover
**Status:** Completed
**Depends on:** K9-0110 (Error Taxonomy + Actionable Failures)

---

## Goal

Improve operator safety and observability by adding:

1. A `preflight` command that validates config, broker connectivity basics, and market data availability without placing orders.
2. A `--dry-run` option for `enter` that executes full selection/validation path and writes logs/journal context without order placement.

## Tasks

### 1. Add `K9 preflight`

Implement a command that checks:

- trade spec load/validate
- broker initialization
- current time/account/quote/chain retrieval

Return `0` on pass, `1` on actionable failure.

### 2. Add `--dry-run` to `K9 enter`

Execute through trade construction and validator checks but skip:

- `place_order`
- order polling
- TP placement

Mark outcome clearly as dry-run in output/log.

### 3. Extend tests/docs

Add integration/smoke tests and update README examples.
