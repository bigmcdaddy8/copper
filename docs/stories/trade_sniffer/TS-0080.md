## Goal

**Scenario 1.5** ‚Äî Place the SIC entry order, wait a configurable number of seconds, check if it is still open, and if so cancel it, reduce the credit by one tick ($0.05), and re-enter. Returns immediately after placing the new order. Answers the FAQ: *"Can we implement a retry loop that brings our required entry price down a few pennies and tries again?"*

**Scenario 2** ‚Äî Inspect the DB and print whether any recently-filled multileg SIC orders were correctly grouped under a single Trade #. This is a verification/read command, not a new order placement.

## Scenario 1.5 Design

```python
def run(
    client: TradierClient,
    account_id: str,
    wait_seconds: int = 30,
    tick_reduction: float = 0.05,
) -> dict:
```

Steps:
1. Call `scenario1.run(client, account_id)` to place the initial entry order
2. Sleep `wait_seconds`
3. Fetch the order from Tradier via `client.get_orders(account_id)` and find the order by ID
4. If status is `filled` ‚Äî return immediately with `{"repriced": False, ...original result}`
5. If status is still `open` (unfilled):
   a. `client.cancel_order(account_id, order_id)`
   b. Reduce credit: `new_credit = max(0.05, original_credit - tick_reduction)`
   c. Re-place the SIC order with `client.place_multileg_order(...)` at the new credit
   d. Return `{"repriced": True, "original_credit": ..., "new_credit": ..., "new_order_id": ...}`

CLI flag: `--wait INTEGER` (default 30 s), `--tick FLOAT` (default 0.05).

## Scenario 2 Design

Read-only. Queries the DB for all `Trade` records and their associated orders via `get_open_trades` + `get_orders_for_trade`. For each trade, prints a summary showing trade_id, status, underlying, and the count + order IDs of linked orders.

## Notes

- `scenario1_5` uses `time.sleep(wait_seconds)` ‚Äî in tests this is patched via `unittest.mock.patch("tradier_sniffer.demo.scenario1_5.time.sleep")`
- The `tick_reduction` floor of $0.05 prevents placing a $0.00 credit order, which Tradier would reject
- `scenario2` is intentionally read-only ‚Äî it is safe to run at any time and will show "no trades" if the DB is empty or the poll loop hasn't detected a fill yet
