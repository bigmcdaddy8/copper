# TS-0020 — Internal Data Models

**Phase:** 2 — Data Models & Persistence (depends on TS-0015)  
**Status:** Done

---

## Goal

Define the canonical Python dataclasses used throughout `tradier_sniffer`. All downstream modules (`db.py`, `engine.py`, demo scenarios) import from `models.py` — this story locks the shape of the data before persistence and engine logic are built. No database or CLI changes in this story; pure data model + tests.

## Models

### Enums

Define these string-valued enums so callers use symbolic names rather than raw strings.

**`OrderStatus`**
```
open | pending | pending_cancel | partially_filled
filled | partially_filled_canceled | canceled | rejected | expired
```
Values match the Tradier `status` field documented in `TRADIER_FAQ.md`.

**`EventType`**
```
new_order | filled | closed | canceled
```
Used by `EventLog.event_type` and by the engine when emitting events.

**`TradeStatus`**
```
open | closed
```

---

### `OrderLeg`

Nested inside `Order` for multileg option orders. Represents one leg of a Tradier multileg order response.

| Field | Type | Notes |
|---|---|---|
| `option_symbol` | `str` | OCC symbol (e.g., `SPX240119P04500000`) |
| `side` | `str` | `buy_to_open` / `sell_to_open` / `buy_to_close` / `sell_to_close` |
| `quantity` | `int` | Contracts |
| `fill_price` | `float \| None` | Per-leg fill; `None` until filled |
| `fill_quantity` | `int \| None` | `None` until filled |

---

### `Order`

Represents a single Tradier order (single-leg or multileg). One row in the `orders` table (TS-0030).

| Field | Type | Notes |
|---|---|---|
| `order_id` | `str` | Tradier order ID |
| `account_id` | `str` | Sandbox account ID |
| `symbol` | `str` | Underlying ticker (e.g., `SPX`) |
| `class_` | `str` | `equity` / `option` / `multileg` / `combo` |
| `order_type` | `str` | `limit` / `market` / `stop` / `stop_limit` |
| `side` | `str` | e.g., `sell_to_open`; for multileg use `None` or `""` |
| `quantity` | `int` | Total contracts / shares |
| `status` | `OrderStatus` | Current order status |
| `duration` | `str` | `day` / `gtc` / `pre` / `post` |
| `limit_price` | `float \| None` | `None` for market orders |
| `fill_price` | `float \| None` | Average fill price; `None` until filled |
| `fill_quantity` | `int \| None` | `None` until filled |
| `option_symbol` | `str \| None` | OCC symbol for single-leg option orders; `None` for equity or multileg |
| `legs` | `list[OrderLeg] \| None` | Non-empty only when `class_ == "multileg"` |
| `tag` | `str \| None` | Tradier tag field; used in TS-0050 for order-to-trade mapping |
| `created_at` | `str` | ISO 8601 UTC |
| `updated_at` | `str \| None` | ISO 8601 UTC; `None` if not yet updated |

---

### `Position`

Represents a single open position returned by `GET /accounts/{id}/positions`. One row per symbol in the `positions` table (TS-0030).

| Field | Type | Notes |
|---|---|---|
| `account_id` | `str` | Sandbox account ID |
| `symbol` | `str` | OCC symbol for options; ticker for equity/ETF |
| `quantity` | `int` | Positive = long, negative = short |
| `cost_basis` | `float` | Total cost basis (not per-share/contract) |
| `date_acquired` | `str` | ISO 8601 date string as returned by Tradier |

---

### `Trade`

The proprietary logical grouping of one or more `Order` objects under a single Trade #. One row in the `trades` table (TS-0030).

| Field | Type | Notes |
|---|---|---|
| `trade_id` | `str` | `TRDS_{#####}_{TTT}` format per GLOSSARY (e.g., `TRDS_00001_NPUT`) |
| `trade_type` | `str` | One of: `SIC`, `PCS`, `CCS`, `NPUT`, `NCALL`, `CCALL`, `STOCK` |
| `status` | `TradeStatus` | `open` on creation; `closed` when all legs exited |
| `underlying` | `str` | Underlying ticker (e.g., `SPX`) |
| `opened_at` | `str` | ISO 8601 UTC — timestamp of first associated order |
| `closed_at` | `str \| None` | ISO 8601 UTC — `None` while trade is open |
| `notes` | `str \| None` | Free-form notes; `None` by default |

**Trade ID format validation helper:** `is_valid_trade_id(trade_id: str) -> bool`
- Pattern: `^TRDS_\d{5,}_[A-Z]+$`
- Used in tests and defensively in `assign_trade()` (TS-0050)

---

### `TradeOrderMap`

Maps a `Trade` to each `Order` that belongs to it, with a `role` label indicating the order's function within the trade lifecycle.

| Field | Type | Notes |
|---|---|---|
| `trade_id` | `str` | FK → `Trade.trade_id` |
| `order_id` | `str` | FK → `Order.order_id` |
| `role` | `str` | `entry` / `tp` / `adjustment` / `exit` |
| `mapped_at` | `str` | ISO 8601 UTC — when the mapping was created |

---

### `EventLog`

Append-only audit trail of all state transitions detected by the engine. One row per event in the `event_log` table (TS-0030).

| Field | Type | Notes |
|---|---|---|
| `event_id` | `int \| None` | Auto-assigned by DB on insert; `None` before persistence |
| `timestamp` | `str` | ISO 8601 UTC |
| `event_type` | `EventType` | `new_order` / `filled` / `closed` / `canceled` |
| `order_id` | `str \| None` | Associated order; `None` for trade-level events |
| `trade_id` | `str \| None` | Associated trade; `None` for order-level events before mapping |
| `details` | `str` | JSON-serialised dict of relevant state snapshot (fill price, old status → new status, etc.) |

## Notes

- `models.py` is pure stdlib — no `httpx`, no `typer`, no `rich`. Downstream modules import from here; it imports from nothing in the app.
- `Trade.trade_type` is a plain `str` in this story; a `TradeType` enum is introduced in TS-0050 where `assign_trade()` logic is built. If it is convenient to define the enum here, that is acceptable and should be noted in the TS-0050 handoff.
- `OrderLeg` is not persisted as its own DB row in TS-0030 — leg data will be serialised as JSON into an `Order` column. This avoids a fourth join table for the common case.
- `EventLog.details` is free-form JSON string rather than a structured type — keeps the model simple and avoids coupling to engine internals that may evolve in TS-0040.
- Field names follow Python conventions (`class_` not `class`); the DB layer in TS-0030 maps `class_` → `order_class` column name to avoid the SQL reserved word.
