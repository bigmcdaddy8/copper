## Tasks

### 1. `config.py` ‚Äî `SnifferConfig`

```python
@dataclass
class SnifferConfig:
    api_key: str
    account_id: str
    poll_interval: int = 10   # seconds between poll cycles
```

- `api_key` and `account_id` sourced from env vars (loaded by CLI layer, not config itself)
- `poll_interval` default of 10 seconds; will be overridable via `--interval` flag in TS-0040

### 2. `tradier_client.py` ‚Äî `TradierClient`

Hardcoded to sandbox (`https://sandbox.tradier.com/v1/`). No production URL ‚Äî this app is sandbox-only by design.

**Constructor:**
```python
TradierClient(api_key: str)
```
- Sets `Authorization: Bearer <api_key>` and `Accept: application/json` headers on all requests
- 30-second request timeout

**Rate-limit throttling** ‚Äî mirrors `trade_hunter` client exactly:
- Reads `X-Ratelimit-Available` and `X-Ratelimit-Expiry` response headers
- Hard throttle: if `available <= 5`, sleep until window resets before next request
- Adaptive delay: spread remaining calls evenly across the remaining window, clamped to `[0.05s, 2.0s]`

**Endpoints:**

| Method | Signature | Tradier endpoint |
|---|---|---|
| `get_user_profile()` | `-> dict` | `GET /user/profile` |
| `get_orders(account_id)` | `-> list[dict]` | `GET /accounts/{id}/orders` |
| `get_positions(account_id)` | `-> list[dict]` | `GET /accounts/{id}/positions` |

- `get_orders` normalises single-order responses (Tradier returns a bare dict when there is exactly one order, a list otherwise) ‚Äî always returns `list[dict]`
- `get_positions` applies the same single-vs-list normalisation
- Non-2xx responses raise `TradierAPIError(status_code, message)`

**Cancel/rejection codes** ‚Äî add a module-level dict documenting known Tradier order status values and rejection reasons (sourced from Tradier API docs during implementation), e.g.:
```python
# Known Tradier order statuses
ORDER_STATUSES = {
    "filled": "Order fully filled",
    "partially_filled": "Order partially filled",
    "open": "Order open/working",
    "expired": "Order expired (Day order past session end)",
    "canceled": "Order canceled by user or broker",
    "pending": "Order received, not yet working",
    "rejected": "Order rejected by broker or exchange",
    ...
}
```
This satisfies the FAQ question for TS-0010.

### 3. `tests/test_client.py`

Use `httpx.MockTransport` (same pattern as `trade_hunter/tests/test_tradier_client.py`):

- Auth header is injected on every request
- `get_user_profile` parses and returns response dict
- `get_orders` returns `[]` when no orders; normalises single-order dict to `[dict]`
- `get_positions` returns `[]` when no positions; normalises single-position dict to `[dict]`
- Non-2xx raises `TradierAPIError` with correct `status_code`
- Rate-limit hard throttle triggers `time.sleep` when `available <= 5`
- Rate-limit hard throttle does NOT sleep when `available > 5`
- Adaptive delay stays within `[MIN_DELAY, MAX_DELAY]`
