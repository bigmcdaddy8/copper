## Goal

Test four edge cases from `TRADIER_FAQ.md` that require specific market conditions: nickel pricing behaviour, option expiration timing, after-hours GTC placement, and after-hours stock quotes. Because these cannot all be exercised in a single interactive session, this story delivers:

1. A `tradier_sniffer demo edge_cases [--run <test>]` CLI command for mechanical tests and checklist printing.
2. Shell scripts in `apps/tradier_sniffer/scripts/` for cron-based daily automation.

---

## Architecture: Cron-based Daily Observation Loop

### Shell scripts (`apps/tradier_sniffer/scripts/`)

| Script | Cron schedule (CT) | What it does |
|---|---|---|
| `daily_entry.sh` | `45 8 * * 1-5` | `tradier_sniffer demo scenario1` ‚Äî places daily SIC entry |
| `expiry_check.sh` | `10 15 * * 1-5` | `tradier_sniffer demo edge_cases --run expiry_timing` |
| `after_hours_check.sh` | `15 15 * * 1-5` | `tradier_sniffer demo edge_cases --run after_hours_quotes` |
| `nickel_test.sh` | Run once manually | `tradier_sniffer demo edge_cases --run nickel_pricing` |
| `cron_setup.sh` | ‚Äî | Prints crontab entries with CT schedule |

Each script sources `.env`, runs the command, and appends output to `logs/edge_cases_YYYY-MM-DD.log`.

**Timezone:** System clock is `America/Chicago` (CT). CT is always 1 hour behind ET (both CST/CDT and EST/EDT shift together), so the offset is stable year-round. No DST adjustment needed between CT and ET.

## Edge Case Sub-tests

### EC-1: Nickel Pricing (`--run nickel_pricing`)

Place a Day STO Limit with a penny-priced limit on a nickel-only option. Observe acceptance, rejection, or silent rounding.

**Observation checklist:**
- [ ] Order rejected ‚Äî `reason_description` = `invalid_price`
- [ ] Order accepted ‚Äî price was silently rounded to nearest nickel
- [ ] Order accepted ‚Äî price as submitted (Tradier does not enforce nickel increment)

### EC-2: Option Expiration Timing (`--run expiry_timing`)

After market close (4:10 PM ET), poll orders/positions for today's 0DTE SPX options.

**Observation checklist:**
- [ ] `get_positions` ‚Äî option still visible at 4:10 PM; time of disappearance: ______
- [ ] `get_orders` ‚Äî status changed to `expired` at: ______
- [ ] Approximate delay from 4:00 PM close to `expired` status: ______ minutes

### EC-3: After-hours GTC Placement (`--run after_hours_gtc`)

After 4:05 PM ET, place a GTC BTC limit order for an open position.

**Observation checklist:**
- [ ] Order accepted ‚Äî status `open`, queued for next session
- [ ] Order rejected ‚Äî `reason_description` = `market_closed`
- [ ] Order accepted ‚Äî status `pending`

### EC-4: After-hours Stock Quotes (`--run after_hours_quotes`)

After 4:05 PM ET, call `GET /markets/quotes?symbols=SPY,QQQ`.

**Observation checklist:**
- [ ] `last` price matches after-hours price in Tradier desktop app
- [ ] `last` is the regular-session closing price (no after-hours data surfaced)
- [ ] `extended_trade` or `post_market` sub-object present in response: yes / no
