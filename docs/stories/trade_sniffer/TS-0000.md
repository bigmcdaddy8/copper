# TS-0000 — App Scaffold & Environment Validation

**Phase:** 0 — Foundation  
**Status:** Done

---

## Goal

Create the `tradier_sniffer` app under `apps/tradier_sniffer/` mirroring the `trade_hunter` workspace structure. The app must install cleanly, expose a working `--help` entry point, and pass smoke tests before any feature work begins.

---

## Tasks

1. Create directory tree:
   ```
   apps/tradier_sniffer/
   ├── pyproject.toml
   ├── src/
   │   └── tradier_sniffer/
   │       ├── __init__.py
   │       └── cli.py
   └── tests/
       └── test_smoke.py
   ```

2. Write `pyproject.toml` — workspace member, entry point `tradier_sniffer = "tradier_sniffer.cli:app"`, deps: `typer`, `rich`, `python-dotenv`, `httpx` (no extras beyond stdlib SQLite)

3. Write `cli.py` — bare `typer.Typer()` app with callback docstring `tradier_sniffer — Tradier sandbox automated trading POC.`; no sub-commands yet

4. Write `test_smoke.py` — invoke `tradier_sniffer --help` via `subprocess` (same pattern as `trade_hunter/tests/test_smoke.py`), assert exit code 0 and `"tradier_sniffer"` in stdout

5. Register in workspace `pyproject.toml` (root-level `[tool.uv.workspace]` members list)

## Acceptance Criteria

- [x] `uv sync --all-groups --all-packages` resolves cleanly
