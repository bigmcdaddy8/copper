## Tasks

### 1. Add endpoints to `tradier_client.py`

Two new methods alongside the existing three:

| Method | Signature | Tradier endpoint |
|---|---|---|
| `get_balances(account_id)` | `-> dict` | `GET /accounts/{id}/balances` |
| `get_history(account_id)` | `-> list[dict]` | `GET /accounts/{id}/history` |

- `get_history` normalises single-item responses to `list[dict]` (same pattern as `get_orders`)

### 2. Wire up `.env` loading in `cli.py`

Add a `@app.callback()` that calls `load_dotenv()` and reads:
- `TRADIER_SANDBOX_API_KEY` (required â€” exit with error if missing)
- `TRADIER_SANDBOX_ACCOUNT_ID` (required â€” exit with error if missing)

Build a `SnifferConfig` and stash it on the Typer context (`ctx.obj`) so sub-commands can access it.

### 3. Add `discover` sub-command to `cli.py`

```
tradier_sniffer discover
```

Calls in order:
1. `GET /user/profile` â€” print account name and account number
2. `GET /accounts/{id}/balances` â€” print a Rich table of all returned fields and their values; flag any needed fields that are absent (net liq, cash, PDT count, option BP)
3. `GET /accounts/{id}/history` â€” print count of history entries returned and the most recent 5 as a table

Output is Rich-formatted to stdout. No flags needed.

### 4. Research and document findings

During implementation, read the Tradier API docs for:
- **Security / auth**: API key scope, OAuth support, IP whitelisting, 2FA â€” document compatibility with automated (non-interactive) access
- **Exchange routing**: available values for the `route` / `duration` order fields; whether a generic BEST routing option exists
- **Account-level risk controls**: any position limits, order size caps, or max-trades settings configurable at account level

Inspect live sandbox responses for:
- **PDT flag**: look for any `day_trades`, `pdt`, or related fields in balances, orders, or history responses
- **Maintenance BPR**: look for `maintenance_margin`, `margin_requirement`, or similar fields in positions/orders responses
