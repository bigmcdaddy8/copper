Body:
```
class=option
option_symbol={option_symbol}
side={side}
quantity={quantity}

duration={duration}

tag={tag}
```

Used for GTC BTC TP: `side="buy_to_close"`, `duration="gtc"`.

### `place_spread_order(account_id, legs, price, duration="day", tag=None) -> dict`
Same as `place_multileg_order` but expects exactly 2 legs. Reuses `place_multileg_order` internally (no code duplication — it's just an alias/thin wrapper for clarity in scenario code).

## Scenario 3 Design

```python
def run(client, account_id, tp_pct=0.50) -> dict:
```

1. Call `scenario1.run(client, account_id)` → `entry`
2. Calculate TP credit: `tp_price = round(entry["credit"] * tp_pct, 2)`
3. Build a single-leg BTC order for each of the 4 short legs (2 short legs of the condor: short put + short call):
   - Actually, a GTC TP for a SIC is a single multileg BTC order (buy to close the 4 legs at the TP price)
   - Use `place_multileg_order` with all 4 legs flipped to `buy_to_close` / `sell_to_close`, `duration="gtc"`, `price=tp_price`
4. Print the TP order ID and instructions: "Stop the poll loop and restart it — reconciliation will detect the TP fill"
5. Return `{"entry_order_id": ..., "tp_order_id": ..., "tp_price": ..., "expiry": ...}`

**Observation note:** After scenario3 places the orders, the user should stop the poll loop (Ctrl-C), wait for the TP to trigger (in sandbox this may require manual fill via Tradier UI), then restart the poll loop. `reconcile()` should detect the TP fill and close the trade.

## Scenario 4 Design

```python
def run(client, conn, account_id) -> dict:
```

1. Call `scenario1.run(client, account_id)` → `entry` (places the SIC)
2. Wait for the fill to be detected via a brief poll of Tradier orders (up to `max_wait_seconds=60`)
   - If not filled within the wait window, return `{"status": "entry_unfilled", ...}` — the user should wait and re-run
3. Once filled, look up the Trade # from the DB via the order ID
4. Place the adjustment: BTC the short put spread (buy back the short put + sell back the long put), then STO a new put spread at strikes 10 points lower — as a single 2-leg multileg order
5. Call `assign_trade(conn, adjustment_order, role="adjustment")` with the existing trade_id
6. Return `{"trade_id": ..., "entry_order_id": ..., "adjustment_order_id": ..., "adjustment_credit": ...}`

**Design note on adjustment:** The adjustment order is a 2-leg multileg: BTC the short put + STO new short put (lower strike). This tests whether Tradier accepts a 2-leg adjustment as a single order (FAQ question). The long put protection may need to be moved separately — scenario4 only rolls the short leg for simplicity.
