## Public API (`db.py`)

| Function | Signature | Notes |
|---|---|---|
| `init_db` | `(db_path: str) -> sqlite3.Connection` | Creates tables + seeds `poll_state`; safe to call on existing DB (uses `CREATE TABLE IF NOT EXISTS`) |
| `upsert_order` | `(conn, order: Order) -> None` | `INSERT OR REPLACE` |
| `insert_trade` | `(conn, trade: Trade) -> None` | `INSERT` — raises `sqlite3.IntegrityError` on duplicate `trade_id` |
| `update_trade_status` | `(conn, trade_id: str, status: TradeStatus, closed_at: str \| None = None) -> None` | Updates `status` and optionally `closed_at` |
| `insert_trade_order_map` | `(conn, mapping: TradeOrderMap) -> None` | `INSERT OR IGNORE` — silently skips duplicates |
| `append_event` | `(conn, event: EventLog) -> int` | Inserts and returns the assigned `event_id` |
| `get_order` | `(conn, order_id: str) -> Order \| None` | Returns hydrated `Order` or `None` |
| `get_trade` | `(conn, trade_id: str) -> Trade \| None` | Returns hydrated `Trade` or `None` |
| `get_open_trades` | `(conn) -> list[Trade]` | All rows where `status = 'open'` |
| `get_orders_for_trade` | `(conn, trade_id: str) -> list[Order]` | Via `trade_order_map` join |
| `get_recent_events` | `(conn, limit: int = 50) -> list[EventLog]` | Ordered by `event_id DESC` |
| `get_poll_state` | `(conn) -> dict[str, str]` | Returns full `poll_state` as `{key: value}` dict |
| `set_poll_state` | `(conn, key: str, value: str) -> None` | `INSERT OR REPLACE` on `poll_state` |
| `next_trade_sequence` | `(conn) -> int` | Reads `trade_sequence`, increments by 1, writes back, returns new value |

## Tasks

### 1. Add `db_path` to `config.py`

```python
db_path: str = "tradier_sniffer.db"
```

### 2. Implement `db.py`

- `init_db` — runs all five `CREATE TABLE IF NOT EXISTS` statements plus `INSERT OR IGNORE` for the two `poll_state` seed rows, then returns the connection with `row_factory = sqlite3.Row`
- Serialisation helpers (module-private):
  - `_order_to_row(order)` — returns a dict mapping dataclass fields to column names; serialises `legs` to JSON; maps `class_` → `order_class`
  - `_row_to_order(row)` — inverse; deserialises `legs_json` back to `list[OrderLeg]`
- All functions accept an open `sqlite3.Connection`; callers manage connection lifecycle

## Notes

- `row_factory = sqlite3.Row` is set on the connection in `init_db` so rows support both index and column-name access
- `next_trade_sequence` must be called inside a transaction if callers need atomicity (engine will handle this in TS-0040)
- No `positions` table — positions are fetched fresh from the API each poll cycle and compared in-memory
- `legs_json` is `NULL` for non-multileg orders, not an empty array — `_row_to_order` handles both
