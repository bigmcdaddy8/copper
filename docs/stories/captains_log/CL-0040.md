# CL-0040 — Query CLI

## Summary
Add `list` and `show` commands to the `captains_log` CLI for querying the trade journal from the terminal.

## Acceptance Criteria
- `captains_log list` exits 0 (even on empty DB)
- `captains_log list --date YYYY-MM-DD --outcome FILLED --spec NAME` all filter correctly
- `captains_log show <trade_id>` exits 0 and shows all fields
- `captains_log show <prefix>` matches on trade_id prefix
- `captains_log show nonexistent` exits 1
- `uv run pytest apps/captains_log -q` passes (4 CLI tests)

## Commands

```
captains_log list [--date YYYY-MM-DD] [--outcome OUTCOME] [--spec SPEC_NAME]
captains_log show <trade-id>
```

`list` renders a Rich Table with columns: ID (8 chars) | Spec | Outcome | Underlying | Expiration | Credit | TP Status | Entered At

`show` renders all 22 fields in a two-column key/value table.
