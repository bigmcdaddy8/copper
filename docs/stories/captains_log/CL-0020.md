# CL-0020 — Journal (SQLite Persistence)

## Summary
Implement the `Journal` class backed by SQLite (stdlib `sqlite3`). Provides write, read, list, and TP-fill-update operations.

## Acceptance Criteria
- DB file auto-created at `data/captains_log/trades.db` (or `CL_DB_PATH` env override)
- Schema auto-created on first instantiation
- `journal.record(trade)` is idempotent (INSERT OR IGNORE on `trade_id`)
- `journal.get_trade(trade_id)` returns `None` for unknown IDs
- `journal.list_trades()` supports optional filters: `date`, `outcome`, `spec_name`
- `journal.update_tp_fill()` sets `tp_fill_price`, `realized_pnl`, `tp_status="FILLED"`
- `uv run pytest apps/captains_log -q` passes (8 journal tests)
