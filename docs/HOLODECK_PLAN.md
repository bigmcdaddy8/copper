# Holodeck — Implementation Plan

## Context

Holodeck is the simulation broker for the Copper automated trading system. It implements the
**Broker Interface Contract (BIC)**, enabling K9 to operate deterministically in a local
environment before touching Tradier sandbox or production.

Holodeck is **not** a backtester and does not aim for market realism. Its job is to let K9
(and later captains_log and encyclopedia_galactica) be built and tested quickly in a
fully self-contained, reproducible environment.

This plan produces two workspace members:

| Package | Path | Purpose |
|---|---|---|
| `bic` | `apps/bic/` | Abstract `Broker` interface + normalized data models (shared lib, no CLI) |
| `holodeck` | `apps/holodeck/` | Simulation broker implementing BIC; minimal CLI for data gen + scenarios |

Both are registered as uv workspace members.

## Phase Map

### Phase 0 — Foundation (BIC Package)

Establish the shared `bic` package that decouples execution logic from any specific broker.
K9 and Holodeck both import from here.

| Story | Title |
|---|---|
| HD-0000 | BIC Data Models |
| HD-0010 | BIC Abstract Broker |

### Phase 1 — Virtual Time & Data

Scaffold the `holodeck` package, implement step-based virtual time, and generate the synthetic
SPX minute-bar dataset that all other components depend on.

| Story | Title |
|---|---|
| HD-0020 | Holodeck Scaffold & VirtualClock |
| HD-0030 | Synthetic SPX Data Generator & MarketDataStore |

### Phase 2 — Option Pricing

Implement the synthetic option chain builder — deterministic pricing and delta approximation
good enough for K9's delta-based strike selection.

| Story | Title |
|---|---|
| HD-0040 | Synthetic Option Pricing & Delta Model |

### Phase 3 — Account & Orders

Build the in-memory account state tracker and the order acceptance/fill engine.

| Story | Title |
|---|---|
| HD-0050 | AccountLedger |
| HD-0060 | OrderEngine |

### Phase 4 — Expiration & Broker Wire-Up

Implement 0DTE position expiration resolution, then wire all components into the concrete
`HolodeckBroker` that fully satisfies the BIC interface.

| Story | Title |
|---|---|
| HD-0070 | ExpirationEngine |
| HD-0080 | HolodeckBroker (BIC Wire-Up) |

### Phase 5 — Deterministic Test Scenarios

Implement the 7 core simulation scenarios and expose them via CLI, providing repeatable
end-to-end validation of Holodeck's behavior.

| Story | Title |
|---|---|
| HD-0090 | Deterministic Test Scenarios |

## Dependencies

### `apps/bic/pyproject.toml`

```toml
[project]
name = "bic"
version = "0.1.0"
requires-python = ">=3.13"
dependencies = []              # stdlib only — no runtime deps

[dependency-groups]
dev = ["pytest>=8"]

[build-system]
requires = ["hatchling>=1.24"]
build-backend = "hatchling.build"

[tool.hatch.build.targets.wheel]
packages = ["src/bic"]
```

### `apps/holodeck/pyproject.toml`

```toml
[project]
name = "holodeck"
version = "0.1.0"
requires-python = ">=3.13"
dependencies = [
    "typer>=0.12",
    "rich>=13",
    "bic",                     # workspace sibling
]

[project.scripts]
holodeck = "holodeck.cli:app"

[dependency-groups]
dev = ["pytest>=8"]

[build-system]
requires = ["hatchling>=1.24"]
build-backend = "hatchling.build"

[tool.hatch.build.targets.wheel]
packages = ["src/holodeck"]
```

No `httpx`, `pandas`, or `python-dotenv` — Holodeck has no network calls, no tabular
data processing, and no secrets.
